# 📚 Dicionário Git do QA

| Termo      | Significado                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------------- |
| **Commit** | Registro de uma alteração realizada no projeto. Cada commit representa um ponto da evolução do código.        |
| **Push**   | Envia os commits da máquina local para o repositório remoto no GitHub.                                        |
| **Pull**   | Atualiza o repositório local trazendo as alterações existentes no repositório remoto.                         |
| **Merge**  | Une alterações de diferentes branches em uma única branch.                                                    |
| **Branch** | Ramificação do projeto utilizada para desenvolver funcionalidades ou correções sem afetar a versão principal. |

# 💻 Comandos Favoritos

```bash
# Clonar um repositório
git clone <url>

# Verificar o status do projeto
git status

# Adicionar arquivos
git add .

# Criar um commit
git commit -m "feat: add initial documentation"

# Enviar alterações
git push origin main

# Atualizar o repositório local
git pull origin main

# Criar uma nova branch
git checkout -b feature/nova-funcionalidade

# Trocar de branch
git checkout main

# Visualizar histórico de commits
git log --oneline

# Ver branches existentes
git branch
```
